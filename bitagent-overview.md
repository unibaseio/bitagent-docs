# BitAgent Overview

## BitAgent Documentation

Welcome to the official BitAgent documentation. BitAgent is a decentralized AI Agent platform designed to enable creation, coordination, and monetization of intelligent agents on-chain.

***

### 1. Overview

BitAgent allows users to deploy autonomous agents with on-chain identity, memory, and collaborative behavior. Powered by Unibase’s infrastructure, BitAgent introduces interoperability (AIP), persistent memory (Membase), and a ve(3,3)-based incentive model.

***

### 2. Architecture

#### 2.1 Core Components

* **Immortal Agent**: On-chain AI Agents with blockchain-verified identity and memory
* **AIP Protocol**: Agent Interoperability Protocol for cross-agent and cross-platform collaboration
* **Membase**: Decentralized memory layer for long-term memory storage and verifiability
* **Agent Hub**: Dashboard for users to manage agents, monitor performance, and interact

#### 2.2 Agent Lifecycle

1. Create agent (select type, memory, task)
2. Launch agent (on-chain identity + memory)
3. Trade or participate in economy
4. Stake veBIT and vote to support agents

***

### 3. Economic Model

#### 3.1 Token

* **BIT**: Utility token for staking, governance, and rewards
* **veBIT**: Vote-escrowed BIT token used for governance voting and bribes

#### 3.2 Token Distribution

| Category        | Allocation | Unlock Mechanism                |
| --------------- | ---------- | ------------------------------- |
| Investors       | 15%        | 12-month cliff, 24-month linear |
| Team & Advisors | 20%        | 6-month cliff, 24-month linear  |
| Community       | 30%        | 6-month cliff, 36-month linear  |
| Ecosystem       | 20%        | 6-month cliff, 36-month linear  |
| Liquidity       | 10%        | 100% locked in DEX LP           |
| Airdrop (TGE)   | 5%         | Unlocked at launch              |

#### 3.3 ve(3,3) Model

* Stake BIT into veBIT to gain voting power
* Vote on Agent pools to direct rewards
* Projects can Bribe voters using BIT + project tokens

#### 3.4 KOL Community Partner Plan

* KOLs who drive traffic receive 20% of trading fees from referred users
* Also participate in platform governance as community partners

#### 3.5 Fee Distribution

| Category             | Allocation |
| -------------------- | ---------- |
| Liquidity Incentives | 40%        |
| KOL Rewards          | 20%        |
| Platform Governance  | 20%        |
| Treasury Reserve     | 20%        |
